# 프로젝트 구조 가이드

OpenClaw은 pnpm 워크스페이스를 활용한 **모노레포** 구조입니다. 이 문서에서는 프로젝트의 전체 디렉토리 레이아웃과 각 모듈의 역할을 설명합니다.

## 최상위 디렉토리 레이아웃

```
openclaw/
  apps/                  # 네이티브 앱 (macOS, iOS, Android)
  assets/                # 정적 리소스 (아이콘, 이미지 등)
  dist/                  # 빌드 결과물 (tsdown 출력)
  docs/                  # 문서 (Mintlify 기반)
  extensions/            # 확장 모듈 (35개 채널 및 기능 확장)
  git-hooks/             # Git 훅 스크립트
  node_modules/          # 의존성 패키지
  packages/              # 내부 패키지 (clawdbot, moltbot)
  patches/               # 의존성 패치 파일
  scripts/               # 빌드 및 유틸리티 스크립트
  skills/                # 스킬 모듈 (52개 AI 도구)
  src/                   # 핵심 소스 코드
  Swabble/               # Swabble 관련 모듈
  test/                  # 통합 테스트
  ui/                    # 웹 UI 컴포넌트
  vendor/                # 벤더 라이브러리
  openclaw.mjs           # CLI 실행 엔트리 포인트
  package.json           # 프로젝트 메타데이터 및 의존성
  pnpm-workspace.yaml    # pnpm 워크스페이스 설정
  tsconfig.json          # TypeScript 컴파일러 설정
  tsdown.config.ts       # 빌드 도구 설정
  vitest.config.ts       # 테스트 프레임워크 설정
```

## src/ 핵심 소스 디렉토리

`src/` 디렉토리는 OpenClaw의 모든 핵심 로직이 위치하는 곳입니다. 기능별로 하위 디렉토리가 구분되어 있습니다.

### AI 에이전트 실행

| 디렉토리      | 설명                                                                                                                                              |
| ------------- | ------------------------------------------------------------------------------------------------------------------------------------------------- |
| `agents/`     | AI 에이전트 실행 엔진. 시스템 프롬프트 구성, AI 모델 호출, 도구 실행, 세션 관리, 응답 스트리밍 처리 등 AI 상호작용의 핵심 로직이 모두 포함됩니다. |
| `providers/`  | LLM 프로바이더 추상화. Claude, OpenAI, Gemini, Bedrock, Ollama 등 각 AI 서비스의 API 호출 로직을 관리합니다.                                      |
| `auto-reply/` | 자동 응답 생성. 메시지에 대한 AI 응답을 생성하고 템플릿을 적용하는 로직입니다.                                                                    |
| `memory/`     | 컨텍스트 메모리 관리. 대화 이력 저장, 검색, 컨텍스트 윈도우 관리 기능을 제공합니다.                                                               |

### 서버 및 네트워크

| 디렉토리    | 설명                                                                                                                                    |
| ----------- | --------------------------------------------------------------------------------------------------------------------------------------- |
| `gateway/`  | Express 5 기반 HTTP/WebSocket 서버. Webhook 수신, OpenAI 호환 API, 인증, 채팅 세션 관리, 브로드캐스트 등 서버의 모든 기능을 포함합니다. |
| `infra/`    | 인프라 유틸리티. 바이너리 관리, 포트 관리, 네트워킹, 환경 변수 처리, 런타임 가드, dotenv 로딩 등 시스템 레벨 기능입니다.                |
| `security/` | 보안 관련 모듈. 인증, 권한 검사, 보안 정책 적용 등을 처리합니다.                                                                        |

### CLI 및 명령

| 디렉토리    | 설명                                                                                                               |
| ----------- | ------------------------------------------------------------------------------------------------------------------ |
| `cli/`      | Commander.js 기반 CLI 프레임워크. 프로그램 빌드, 명령 등록, 의존성 주입, 프로파일 관리 등 CLI 인프라를 담당합니다. |
| `commands/` | CLI 명령 구현체. 각 하위 명령(send, config, status 등)의 실제 실행 로직이 위치합니다.                              |

### 채널 통합

내장 채널 어댑터와 공통 채널 로직이 분리되어 있습니다.

| 디렉토리    | 설명                                                                                                                         |
| ----------- | ---------------------------------------------------------------------------------------------------------------------------- |
| `channels/` | 채널 공통 로직. 채널 레지스트리, 설정, allowlist, 멘션 감지, 대화 유형 판별, 타이핑 표시 등 모든 채널이 공유하는 기능입니다. |
| `telegram/` | Telegram Bot API 어댑터. Telegram 메시지 수발신, 인라인 쿼리, 리액션 처리 등을 구현합니다.                                   |
| `discord/`  | Discord Bot 어댑터. Discord 메시지, 슬래시 명령, 스레드, 임베드 처리를 구현합니다.                                           |
| `slack/`    | Slack Bot 어댑터. Slack Events API, Block Kit 메시지, 슬래시 명령을 처리합니다.                                              |
| `signal/`   | Signal 메신저 어댑터. signal-cli를 통한 Signal 프로토콜 통합입니다.                                                          |
| `imessage/` | iMessage 어댑터. macOS의 Messages 앱과 연동합니다.                                                                           |
| `whatsapp/` | WhatsApp 어댑터. WhatsApp Business API 또는 Web 클라이언트를 통한 연동입니다.                                                |
| `web/`      | 웹 채널 어댑터. 브라우저 기반 채팅 인터페이스를 제공합니다.                                                                  |
| `line/`     | LINE 메신저 어댑터. LINE Messaging API를 통한 메시지 처리입니다.                                                             |

### 설정 및 스키마

| 디렉토리    | 설명                                                                                                                    |
| ----------- | ----------------------------------------------------------------------------------------------------------------------- |
| `config/`   | Zod 4 기반 설정 시스템. YAML/JSON 설정 파일 로딩, 스키마 검증, 에이전트별 설정, 채널별 설정, 경로 관리 등을 처리합니다. |
| `sessions/` | 세션 스토리지 관리. 대화 세션의 저장, 조회, 키 관리 기능입니다.                                                         |

### 확장 기능

| 디렉토리      | 설명                                                                                                                                                       |
| ------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `plugins/`    | 플러그인 런타임. jiti를 사용한 동적 TypeScript 플러그인 로딩, 라이프사이클 관리, 샌드박싱을 처리합니다.                                                    |
| `plugin-sdk/` | 플러그인 공개 API. 확장 및 스킬 개발자가 사용하는 타입 정의와 유틸리티를 제공합니다. `package.json`의 exports에서 `openclaw/plugin-sdk`로 접근 가능합니다. |
| `hooks/`      | 이벤트 시스템. 메시지, 도구, 세션 등의 이벤트를 발행하고 구독하는 메커니즘입니다.                                                                          |

### 미디어 및 브라우저

| 디렉토리               | 설명                                                                                     |
| ---------------------- | ---------------------------------------------------------------------------------------- |
| `browser/`             | Playwright 기반 브라우저 자동화. 웹 페이지 스크린샷, 콘텐츠 추출 등의 기능을 제공합니다. |
| `media/`               | 미디어 처리. 이미지 변환, PDF 파싱, 파일 업로드/다운로드 등 멀티미디어 관련 기능입니다.  |
| `media-understanding/` | 미디어 이해 모듈. AI를 활용한 이미지, 문서 분석 기능입니다.                              |
| `link-understanding/`  | URL 콘텐츠 분석. 링크의 내용을 파싱하고 요약하는 기능입니다.                             |

### 부가 기능

| 디렉토리    | 설명                                                                         |
| ----------- | ---------------------------------------------------------------------------- |
| `routing/`  | 메시지 라우팅. 수신 메시지를 적절한 에이전트와 세션으로 전달하는 로직입니다. |
| `logging/`  | tslog 기반 구조화 로깅. 로그 레벨, 포맷, 출력 대상 관리를 담당합니다.        |
| `cron/`     | 예약 작업. 정기적으로 실행되는 태스크(세션 정리, 상태 체크 등)를 관리합니다. |
| `tui/`      | 터미널 UI. 터미널에서의 대화형 인터페이스를 제공합니다.                      |
| `tts/`      | Text-to-Speech 기능. 텍스트를 음성으로 변환하는 모듈입니다.                  |
| `markdown/` | Markdown 처리 유틸리티. 메시지 포매팅과 변환 기능입니다.                     |
| `process/`  | 프로세스 관리. 자식 프로세스 실행, 브릿지, exec 유틸리티를 제공합니다.       |
| `shared/`   | 공유 유틸리티. 여러 모듈에서 공통으로 사용하는 타입과 함수입니다.            |
| `types/`    | 전역 타입 정의. 프로젝트 전반에서 사용하는 TypeScript 타입입니다.            |
| `acp/`      | Agent Communication Protocol 관련 모듈입니다.                                |
| `macos/`    | macOS 전용 기능. AppleScript 연동, 시스템 통합 등을 처리합니다.              |
| `pairing/`  | 디바이스 페어링. 네이티브 앱과 서버 간 연결 설정을 관리합니다.               |
| `wizard/`   | 초기 설정 마법사. 사용자를 안내하는 대화형 설정 흐름입니다.                  |
| `terminal/` | 터미널 유틸리티. 터미널 출력 포매팅, 색상 등을 관리합니다.                   |
| `polls.ts`  | 투표 기능. 채널에서 설문조사를 생성하고 관리합니다.                          |

## 파일 탐색 팁

프로젝트에서 원하는 코드를 빠르게 찾기 위한 팁입니다.

**채널 관련 코드를 찾을 때**: 먼저 `src/channels/registry.ts`에서 채널 등록 목록을 확인하고, 해당 채널의 디렉토리(`src/telegram/`, `extensions/telegram/` 등)로 이동하세요.

**AI 모델 관련 코드를 찾을 때**: `src/agents/model-selection.ts`에서 모델 선택 로직을, `src/agents/pi-embedded-runner.ts`에서 실행 로직을 확인하세요.

**설정 스키마를 확인할 때**: `src/config/` 디렉토리에서 Zod 스키마 정의를 찾을 수 있습니다. 설정 키 이름으로 grep 검색하면 빠르게 찾을 수 있습니다.

**테스트를 실행할 때**: 각 소스 파일 옆에 `.test.ts` 파일이 함께 위치합니다. `vitest`로 특정 파일이나 패턴을 지정하여 실행할 수 있습니다.

```bash
# 전체 단위 테스트
pnpm test

# 특정 파일 테스트
pnpm vitest run src/agents/model-selection.test.ts

# 패턴 매칭 테스트
pnpm vitest run --grep "channel"
```
