## Config 시스템 개요

OpenClaw의 모든 동작은 설정 파일로 제어됩니다.
`src/config/` 디렉토리에 130개의 파일이 있으며, YAML/JSON 파일을 Zod 스키마로 검증하여
타입 안전한 설정을 보장합니다.

설정 하나 잘못되면 채널이 연결되지 않거나 에이전트가 동작하지 않을 수 있으므로,
엄격한 검증이 핵심입니다.

## 설정 파일 구조

OpenClaw 설정은 YAML 또는 JSON 형식의 단일 파일입니다.

```yaml
# config.yaml 기본 구조
identity:
  name: "My Assistant"
  systemPrompt: "You are a helpful assistant."

telegram:
  token: "BOT_TOKEN"
  allowFrom:
    - 123456789

discord:
  token: "DISCORD_BOT_TOKEN"
  allowFrom:
    - "user-id"

models:
  default: "claude-sonnet-4-20250514"

agents:
  default:
    model: "claude-sonnet-4-20250514"
    tools:
      bash:
        enabled: true
      web:
        enabled: true

gateway:
  port: 18789
  password: "secret"

session:
  backend: "sqlite"

hooks:
  enabled: true
  token: "webhook-secret"
```

## 핵심 모듈 구조

### 파일 맵

| 파일/디렉토리             | 역할                                                         |
| ------------------------- | ------------------------------------------------------------ |
| `config.ts`               | 설정 로딩, 파일 읽기/쓰기, 마이그레이션의 진입점 (re-export) |
| `io.ts`                   | 실제 설정 파일 I/O 처리. `loadConfig()`, `writeConfigFile()` |
| `validation.ts`           | Zod 스키마를 사용한 설정 객체 검증                           |
| `zod-schema.ts`           | 최상위 Zod 스키마 조합                                       |
| `zod-schema.agents.ts`    | 에이전트 관련 스키마                                         |
| `zod-schema.providers.ts` | 채널/프로바이더 스키마                                       |
| `zod-schema.core.ts`      | 모델, 색상 등 공통 스키마                                    |
| `zod-schema.hooks.ts`     | 웹훅 설정 스키마                                             |
| `zod-schema.session.ts`   | 세션 및 명령어 스키마                                        |
| `zod-schema.approvals.ts` | 실행 승인 스키마                                             |
| `sessions/`               | SQLite 기반 세션 저장소                                      |
| `paths.ts`                | 설정 파일 경로 해석                                          |
| `legacy-migrate.ts`       | 레거시 설정 자동 마이그레이션                                |

### 설정 로딩 흐름

```
config.yaml
    |
    v
io.ts (readConfigFileSnapshot)
    |
    v
JSON5 파싱
    |
    v
validation.ts (validateConfigObject)
    |
    v
zod-schema.ts (OpenClawSchema.parse)
    |
    v
OpenClawConfig 타입 객체
```

## Zod 스키마 패턴

OpenClaw는 모든 설정 검증에 Zod를 사용합니다.
스키마가 여러 파일로 분리되어 있으며, 최종적으로 `zod-schema.ts`에서 결합됩니다.

### 스키마 분리 구조

```
zod-schema.ts (최상위 OpenClawSchema)
  +-- zod-schema.core.ts (ModelsConfigSchema, HexColorSchema)
  +-- zod-schema.agents.ts (AgentsSchema, AudioSchema, BindingsSchema)
  +-- zod-schema.providers.ts (ChannelsSchema)
  +-- zod-schema.hooks.ts (HookMappingSchema, HooksGmailSchema)
  +-- zod-schema.session.ts (SessionSchema, CommandsSchema)
  +-- zod-schema.approvals.ts (ApprovalsSchema)
  +-- zod-schema.agent-runtime.ts (ToolsSchema)
```

### 스키마 작성 예시

실제 코드에서 사용되는 패턴입니다.

```typescript
import { z } from "zod";

// 메모리 경로 스키마
const MemoryQmdPathSchema = z
  .object({
    path: z.string(),
    name: z.string().optional(),
    pattern: z.string().optional(),
  })
  .strict();

// 세션 스키마
const MemoryQmdSessionSchema = z
  .object({
    enabled: z.boolean().optional(),
    exportDir: z.string().optional(),
    retentionDays: z.number().int().nonnegative().optional(),
  })
  .strict();
```

`.strict()`를 사용하여 정의되지 않은 필드가 있으면 즉시 오류를 발생시킵니다.
이는 설정 오타를 조기에 발견하는 데 매우 효과적입니다.

### 타입 추론

Zod 스키마에서 TypeScript 타입을 자동 추론합니다.

```typescript
// 스키마에서 타입 추출
type OpenClawConfig = z.infer<typeof OpenClawSchema>;

// 이 타입이 코드베이스 전체에서 사용됨
function loadConfig(): Promise<OpenClawConfig> {
  /* ... */
}
```

스키마를 수정하면 타입이 자동으로 업데이트되어,
설정 구조 변경 시 관련 코드에서 즉시 타입 오류가 발생합니다.

## 세션 저장소

`src/config/sessions/` 디렉토리에서 SQLite 기반 세션 관리를 담당합니다.

| 파일              | 역할                   |
| ----------------- | ---------------------- |
| `store.ts`        | 세션 CRUD 및 pruning   |
| `transcript.ts`   | 대화 기록 저장/조회    |
| `metadata.ts`     | 세션 메타데이터 관리   |
| `paths.ts`        | 세션 DB 파일 경로 결정 |
| `main-session.ts` | 메인 세션 키 해석      |
| `group.ts`        | 그룹 세션 관리         |
| `reset.ts`        | 세션 초기화            |

세션 데이터는 SQLite 파일에 저장되므로 별도 데이터베이스 서버가 필요 없습니다.
대화 기록 보존 기간은 `retentionDays`로 설정할 수 있습니다.

## 동적 설정 리로드

Gateway가 실행 중일 때 설정 파일을 수정하면, 서버를 재시작하지 않고도
변경 사항이 반영됩니다.

`src/gateway/config-reload.ts`의 `startGatewayConfigReloader()`가 이를 처리합니다.

리로드 과정은 다음과 같습니다.

1. 파일 시스템 감시자가 설정 파일 변경을 감지합니다
2. 변경된 파일을 읽고 Zod 스키마로 검증합니다
3. 검증 성공 시 새 설정을 적용합니다
4. 채널 매니저, 에이전트 설정, 플러그인 상태를 업데이트합니다
5. 검증 실패 시 이전 설정을 유지하고 오류를 로깅합니다

이 메커니즘 덕분에 채널 토큰 변경, allowlist 수정, 모델 교체 등을
서비스 중단 없이 수행할 수 있습니다.

## 레거시 마이그레이션

설정 포맷이 변경될 때 `legacy-migrate.ts`가 자동 마이그레이션을 수행합니다.
이전 버전의 설정 파일도 최신 스키마에 맞게 변환되므로,
업데이트 후 수동으로 설정을 수정할 필요가 없습니다.

## 환경 변수 오버라이드

`runtime-overrides.ts`에서 환경 변수를 통한 설정 오버라이드를 지원합니다.
설정 파일을 수정하지 않고도 특정 값을 변경할 수 있어,
Docker 배포나 CI/CD 환경에서 유용합니다.

## 핵심 요약

Config 시스템은 OpenClaw의 안전 장치입니다.
Zod 스키마로 모든 설정을 엄격히 검증하고,
타입 추론으로 코드 수준의 안전성을 보장합니다.
SQLite 세션, 동적 리로드, 레거시 마이그레이션까지
운영에 필요한 모든 설정 인프라를 제공합니다.
