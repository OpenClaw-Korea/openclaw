# Development Setup

OpenClaw 개발을 시작하기 위한 환경 설정 가이드입니다.
이 문서를 따라가면 로컬에서 프로젝트를 빌드하고 실행할 수 있습니다.

## Prerequisites

개발을 시작하기 전에 아래 도구들이 설치되어 있어야 합니다.

| 도구    | 최소 버전 | 확인 명령어     |
| ------- | --------- | --------------- |
| Node.js | 22+       | `node -v`       |
| pnpm    | 10.23+    | `pnpm -v`       |
| git     | latest    | `git --version` |

<Note>
  OpenClaw은 Node.js 22 이상에서만 동작합니다. nvm 또는 fnm을 사용해 Node 버전을 관리하는 것을
  권장합니다.
</Note>

## 설치 및 실행

<Steps>

<Step title="저장소 클론">
GitHub에서 OpenClaw 저장소를 클론합니다.

```bash
git clone https://github.com/openclaw/openclaw.git
cd openclaw
```

</Step>

<Step title="의존성 설치">
pnpm을 사용해 모든 워크스페이스의 의존성을 설치합니다.

```bash
pnpm install
```

모노레포 구조이므로 pnpm workspaces가 모든 패키지의 의존성을 한 번에 처리합니다.

</Step>

<Step title="환경 변수 설정">
프로젝트 루트의 `.env.example` 파일을 참고해 `.env` 파일을 생성합니다.

```bash
cp .env.example .env
```

OpenClaw은 dotenv를 사용해 `.env` 파일을 자동으로 로드합니다.
API 키 등 필요한 값을 채워넣으세요.

</Step>

<Step title="프로젝트 빌드">
전체 프로젝트를 빌드합니다. tsdown 빌드, plugin-sdk 생성, 에셋 복사가 순서대로 실행됩니다.

```bash
pnpm build
```

</Step>

<Step title="개발 모드 실행">
개발 모드로 OpenClaw을 실행합니다.

```bash
pnpm dev
```

또는 CLI 명령어를 직접 실행할 수도 있습니다.

```bash
pnpm openclaw ...
```

</Step>

</Steps>

## 주요 개발 명령어

### Gateway 개발

Gateway 서버를 watch 모드로 실행합니다. 파일 변경 시 자동으로 재시작됩니다.

```bash
pnpm gateway:dev
```

### 타입 체크

TypeScript 5.9의 타입 검사를 실행합니다. `tsgo`는 네이티브 TypeScript 타입체커입니다.

```bash
pnpm tsgo
```

### 코드 품질

린트와 포맷을 한 번에 확인합니다. oxfmt(포매터)과 oxlint(린터)가 함께 실행됩니다.

```bash
pnpm check
```

포맷 자동 수정이 필요한 경우 아래 명령어를 사용합니다.

```bash
pnpm format:fix
```

## 프로젝트 구조 요약

OpenClaw은 pnpm workspaces 기반 모노레포입니다. 주요 기술 스택은 다음과 같습니다.

| 항목          | 기술                         |
| ------------- | ---------------------------- |
| 언어          | TypeScript 5.9 (ESM, strict) |
| 런타임        | Node.js 22+                  |
| 패키지 매니저 | pnpm 10.23+                  |
| 웹 프레임워크 | Express 5                    |
| 스키마 검증   | Zod 4                        |
| 테스트        | Vitest                       |
| 린터          | oxlint                       |
| 포매터        | oxfmt                        |

<Tip>
빌드나 설치에 문제가 있다면 `node_modules`를 삭제하고 `pnpm install`을 다시 실행해 보세요.

```bash
rm -rf node_modules
pnpm install
```

그래도 해결되지 않는다면 Discord `#setup-help` 채널에서 도움을 받을 수 있습니다.

</Tip>
